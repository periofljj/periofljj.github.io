你不知道的JavaScript读书笔记

Nov 7, 2017 | Hits

编译

js中的作用域，指的是为了储存管理变量而定义的一套规则。

js作为一种编译语言，它与一些传统的编译语言不同，并非是提前编译的。我们在提到“编译”这个行为的时候，通常有这样三个步骤。

  1. 词法分析
  2. 语法分析，生成抽象语法树
  3. 生成可执行的代码

在这个过程中有三个关键角色各司其职

引擎： 从头到尾负责整个编译的过程

编译器： 负责2.3步骤

作用域：

  1. 收集维护变量
  2. 查找变量
  3. 定义规则确定变量访问权限

那么在var a = 2这个简单的声明执行的过程中，它们又是如何配合工作的呢？

1.编译器首先向作用域确认是否有这个变量存在于当前作用域集合中

  • 有，忽略。
  • 没有，在当前作用域集合中创建这个变量。

2.编译器生成可执行的代码给引擎，用于处理“赋值”这个动作。引擎在作用域中逐级查询这个变量（因为存在作用域嵌套）。

  • 找到，为它赋值
  • 没找到，抛出referenceError。
顺便说明引擎查询分为两类：LHS（赋值）与RHS（溯源）

闭包

闭包，作为js中很重要的概念，对其进行阐述。

将内部函数传递到所在的词法作用域之外，内部函数仍持有对原始定义作用域的引用。

例如：B函数嵌套在A函数内部，将B函数作为A函数的返回值。B函数持有对A函数作用域的引用，这个引用叫做闭包。并且A函数的作用域不会被垃圾回收，而是停留在内存中。

模块 是js利用闭包的一个很好的例子。

模块模式需要具备两个必要条件：

  1. 必须有外部的封闭函数，该函数必须至少被调用一次(每次调用都会创建一个新的模块实例)。
  2. 封闭函数必须返回至少一个内部函数，这样内部函数才能在私有作用域中形成闭包，并且可以访问或者修改私有的状态。
